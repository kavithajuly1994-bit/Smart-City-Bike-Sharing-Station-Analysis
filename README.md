🚲 Analyzing Smart City Bike Sharing Data – Power BI

I recently completed a project where I analyzed bike sharing station data to understand availability, utilization, and operational performance across stations using Microsoft Power BI.

📌 Project Purpose

• Analyze bike station performance and availability across different locations.

• Identify high-demand and underutilized stations.

• Understand bike availability trends over time to support better operational planning.

🛠 Tools & Techniques

• Power BI (Data Modeling & Dashboard )

• Power Query for Data Cleaning & Transformation

• DAX for Calculated Columns and Measures

• Data Visualization using Bar Charts, Column Charts, Line Charts, Map, Gauge, and Matrix

📊 KPIs Analyzed

• Total Stations

• Occupied Stands

• Average Bike Availability

• Station Utilization %

🔍 Key Insights

• The bike sharing network consists of 1,028 stations, showing a large urban mobility infrastructure.

• Average station utilization is around 46%, indicating moderate usage of station capacity.

• Bike availability varies significantly across stations, highlighting uneven distribution of bikes.

• Some stations frequently experience empty conditions, indicating higher demand in those locations.

• Trend analysis shows a significant increase in bike availability in 2025, suggesting network expansion.

⚙️ Slicers Implemented

• Station Name

• Station Load Level

• Year

These slicers allow interactive filtering to explore station performance and trends across different time periods.

⚡ Challenges Faced

• Handling missing values and cleaning inconsistent data during transformation.

• Correctly mapping station locations using latitude and longitude coordinates.

• Designing clear visuals that effectively communicate insights from multiple metrics.

📈 Outcome

This dashboard provides a comprehensive view of bike station operations, helping stakeholders monitor availability, identify demand patterns, and optimize bike distribution across the network.
